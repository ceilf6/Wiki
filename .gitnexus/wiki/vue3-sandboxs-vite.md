# sandboxs-vite ‚Äî Vue 3 Sandbox Application

A Vue 3 learning sandbox built with Vite, demonstrating core Vue 3 concepts including the Composition API, Pinia state management, Vue Router, async components, and custom composables.

## Project Overview

This project serves as a hands-on playground for Vue 3 features. It is **not** a production application ‚Äî it is a collection of isolated test components, composables, and patterns that illustrate how Vue 3 APIs work in practice.

### Tech Stack

| Dependency | Purpose |
|---|---|
| `vue@^3.0.4` | Core framework |
| `vue-router@^4.0.13` | Client-side routing |
| `pinia@^3.0.4` | State management (Vuex successor) |
| `vite@^1.0.0-rc.13` | Dev server and bundler |

### Scripts

```bash
npm run dev    # Start Vite dev server (native ESM, no bundling)
npm run build  # Production build via Vite
```

## Application Entry Point

**`src/main.js`** bootstraps the Vue application:

```js
const app = createApp(App)
app.use(router)
app.use(createPinia())
app.mount('#app')
```

Three plugins are registered in order:
1. **Vue Router** ‚Äî client-side navigation
2. **Pinia** ‚Äî global state management

`App.vue` currently renders the `<PiniaTest />` component. Other test components (`Page`, `RenderTriggered`) are commented out and can be swapped in for experimentation.

## Routing

Defined in `src/router/index.js` using `createRouter` with HTML5 history mode. Routes are configured in `src/router/routes.js`:

| Path | Component |
|---|---|
| `/` | `views/Home/index.vue` |
| `/about` | `views/About/index.vue` |

Routes use **async component loading** via the `syncGetComp` utility (despite the name, it wraps `defineAsyncComponent` with a simulated 3-second delay, loading spinner, and error fallback).

## Composables (`src/compositions/`)

Composables are the primary pattern for extracting and reusing stateful logic. Each composable is a function that returns reactive state and methods.

### `useCount`

A minimal counter composable demonstrating `ref` basics.

```js
const { countRef, handleIncrease, handleDecrease } = useCount(0)
```

### `useTodoList`

Loads todos from `localStorage` on initialization and sets up a `watchEffect` to persist changes automatically.

```js
const { todoRef } = useTodoList()
// Any mutation to todoRef.value triggers saveTodos()
```

### `useNewTodo(todoRef)`

Provides a text input binding (`newTodoRef`) and an `addTodo` function that appends new items with a unique ID generated by `generateID()`.

### `useEditTodo(todoRef)`

Handles inline editing of todo items:
- `handleDBLClick(todo)` ‚Äî enters edit mode, caches original title
- `doneEdit(todo)` ‚Äî commits the edit (or removes the item if title is empty)
- `cancelEdit(todo)` ‚Äî restores the cached title
- `allDoneRef` / `setAllChecked(checked)` ‚Äî computed property and setter for "mark all complete"

### `useFilter(todoRef)`

Syncs the active filter (`all`, `active`, `completed`) with the URL hash (`#/all`, `#/active`, `#/completed`). Listens to `hashchange` events and provides computed filtered/remaining/completed counts.

### `useRemoveTodo(todosRef)`

Provides `remove(todo)` for single-item deletion and `removeCompleted()` to clear all completed items.

### `useUser`

A global reactive user state module that simulates Vuex-style state management using Vue 3's `reactive` and `readonly`:

```js
const { user, loading, login, loginOut, whoAmI } = useUser()
```

The internal `userState` is a module-level `reactive` object. A `readonly` wrapper is exposed externally via `toRefs`, so consumers get reactive refs but cannot mutate state directly. All mutations go through the exported async methods (`login`, `loginOut`, `whoAmI`), which call the simulated API in `src/api/user.js`.

### `useModal`

A simple boolean toggle for modal visibility:

```js
const { ModalVisible } = useModal()
```

## Pinia Stores (`src/store/`)

Two stores demonstrate both Pinia API styles.

### `useCounterStore` ‚Äî Options API Style

```js
const store = useCounterStore()
store.increment()
store.asyncIncrement()  // 1-second delay
store.$reset()          // resets state to initial values
store.$patch({ num: 5 }) // batch state updates
```

Uses `storeToRefs` in components to destructure reactive state:

```js
const { num, doubleNum } = storeToRefs(store)
```

### `useTodoListStore` ‚Äî Composition API Style

```js
const store = useTodoListStore()
store.addItem("New task")
```

Uses `reactive` internally and returns state + methods from the setup function.

## Components

### `CheckEditor` (`components/Checkbox/CheckEditor.vue`)

A custom checkbox + text input component demonstrating **custom `v-model` modifiers**:

```vue
<CheckEditor v-model:title.trim="titleVal" v-model:checked="checkedVal" />
```

The component reads `props.titleModifiers` and conditionally trims input. It emits `update:checked` and `update:title` events for two-way binding.

### `Todos` (`components/Todos.vue`)

A complete todo application that composes multiple composables in its `setup()`:

```js
setup() {
  const { todoRef } = useTodoList()
  return {
    todoRef,
    ...useNewTodo(todoRef),
    ...useFilter(todoRef),
    ...useEditTodo(todoRef),
    ...useRemoveTodo(todoRef),
  }
}
```

### `Modal` (`components/Modal/index.vue`)

A full-screen overlay component designed for use with Vue 3's `<Teleport>`:

```vue
<Teleport to="body">
  <Modal v-if="ModalVisible">
    <button @click="ModalVisible = false">Close</button>
  </Modal>
</Teleport>
```

### `Loading` / `ErrorComp`

Used as fallback components for async component loading states.

## Async Components

The project demonstrates three approaches to `defineAsyncComponent`:

**1. Simple import** (in `routes.js` via `syncGetComp`):
```js
defineAsyncComponent({
  loader: async () => {
    await new Promise(resolve => setTimeout(resolve, 3000))
    return await import(path)
  },
  loadingComponent: Loading,
  errorComponent: { render() { return h(ErrorComp, errorMsg) } }
})
```

**2. Inline in `views/Home/index.vue`** ‚Äî `Block3` throws an error to demonstrate the error boundary; `AsyncBlock5` demonstrates the loading state with a 3-second delay.

**3. The `syncGetComp` utility** (`src/utils/syncGetComp.js`) ‚Äî a reusable factory that wraps `defineAsyncComponent` with configurable error messages.

## Utilities (`src/utils/`)

| Function | File | Purpose |
|---|---|---|
| `delay(ms)` | `delay.js` | Returns a Promise that resolves after `ms` milliseconds |
| `syncGetComp(path, errorMsg)` | `syncGetComp.js` | Creates an async component with loading/error states |
| `fetchTodos()` | `todoStorage.js` | Reads todos from `localStorage` |
| `saveTodos(todos)` | `todoStorage.js` | Persists todos to `localStorage` |
| `generateID()` | `todoStorage.js` | Generates a unique ID from timestamp + random hex |
| `filter(todos, filter)` | `todoStorage.js` | Filters a todo array by status (`all`, `active`, `completed`) |

## Simulated API (`src/api/user.js`)

All functions return Promises with a 1-second artificial delay:

- **`login(loginId, loginPwd)`** ‚Äî authenticates against hardcoded credentials (`ceilf6` / `666666`), stores user in `localStorage`
- **`loginOut()`** ‚Äî removes user from `localStorage`
- **`whoAmI()`** ‚Äî restores session from `localStorage` (simulates JWT token check)

## Vite Configuration

`vite.config.js` defines a single path alias:

```js
alias: {
  '@': path.resolve(__dirname, './src')
}
```

**Note:** Vite requires explicit `.vue` file extensions in imports ‚Äî unlike webpack, it does not auto-resolve them.

## Key Vue 3 Patterns Demonstrated

1. **Composition API** ‚Äî all composables use `ref`, `computed`, `watchEffect`, and lifecycle hooks (`onMounted`, `onUnmounted`)
2. **Custom v-model** ‚Äî `CheckEditor` shows multi-model binding with modifiers
3. **`<Teleport>`** ‚Äî the Modal is teleported to `<body>`
4. **Async components** ‚Äî `defineAsyncComponent` with loading/error boundaries
5. **Global state without Vuex** ‚Äî `useUser` uses `reactive` + `readonly` + closures
6. **Pinia** ‚Äî both Options API and Composition API store styles
7. **`storeToRefs`** ‚Äî required when destructuring Pinia store state to preserve reactivity
