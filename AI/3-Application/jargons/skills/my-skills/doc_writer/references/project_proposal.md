# 项目方案模板

## 适用场景

适用于项目立项、执行方案、阶段性推进方案、产品落地方案、活动实施方案等需要说明背景、目标、路径与风险的场景。

## 使用要求

- 方案必须说明“为什么做、做什么、怎么做、谁来做、何时完成、有哪些风险”。
- 若用户只提供主题，优先补全结构，不补造具体业务数据。
- 若涉及预算、周期、责任人等关键要素，缺失时使用占位符。
- 方案正文应强调可执行性，而不是泛泛描述愿景。

## 文风要求

- 用语正式、克制、明确。
- 少写空话，优先使用“负责”“完成”“交付”“验收”“推进”等可执行表达。
- 若为汇报型方案，可适度增强总结和价值表达；若为执行型方案，应更强调时间与责任。
