# 会议纪要模板

## 适用场景

适用于项目例会、部门同步会、需求评审会、复盘会、跨团队沟通会等需要记录讨论过程、结论与后续动作的场景。

## 使用要求

- 先写清会议基本信息，再写讨论过程。
- 讨论内容要按议题归类，不要流水账式堆叠发言。
- 决议事项必须明确结论。
- 待办事项必须尽量写到负责人和时间。
- 若信息缺失，使用占位符补齐，不得臆造事实。

## 文风要求

- 使用客观、中性、正式表达。
- 以“会议讨论认为”“会议决定”“与会人员一致认为”等表达归纳结论。
- 避免口语化记录，如“大家觉得差不多可以”“这个回头再说”。
